--- layout: post title: "DNF performance boost" date: 2017-11-23 14:00:00 +0100 author: Jaroslav Mracek visible: 1 categories: dnf --- Since the last blog there were several releases that enhance functionality and mostly performance especially visible with multiple package arguments. Just try "dnf --assumeno reinstall $(rpm -qa)" with libdnf-0.9.0 and libdnf-0.11.1 and differences is easily visible. But the real huge improvement can be seen when results are compared to YUM.

DNF-2.7.5

  • Enhanced performance for excludes and includes handling
  • Solved several memory leaks
  • Additionally 8 bug reports solved
  • Detailed information about DNF releases

DNF-PLUGINS-CORE-2.1.5

  • Added four new options for list subcommand of copr plugin
  • Additionally 2 bug reports solved
  • Detailed information about DNF-PLUGINS-CORE releases

DNF-PLUGINS-EXTRAS-2.0.4

  • Fixed several issues of system-upgrade plugin
  • Detailed information about DNF-PLUGINS-EXTRAS releases

LIBDNF-0.11.1

  • Huge performance boost for queries with arch and name filters (20x boost)
  • Additionally 4 bug reports solved
